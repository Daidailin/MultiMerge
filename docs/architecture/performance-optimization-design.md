## 三、架构设计（已验证部分）

### 3.1 分级处理策略 ✅

```
A 类（单增+均匀）→ 顺序推进主路径
B 类（单增+不均匀）→ 顺序+局部校正
C 类（乱序）→ 索引+二分查找
```

### 3.2 流式处理架构 ✅

**核心思想**：
- 不一次性加载全部数据
- 逐行读取 → 处理 → 写入输出
- 只保留必要的索引信息

**内存优势（理论上）**：
- 原方案：2GB 文件 → 6-8GB 内存
- 流式处理：2GB 文件 → 约 200MB 内存（仅索引）

**⚠️ 待实测验证**

## 六、实施原则

### 6.1 当前原则

1. **先稳后快**：先保证正确性，再追求性能
2. **渐进式优化**：A 类→B 类→C 类，逐步覆盖
3. **回退机制**：任何时候都有备用方案
4. **数据驱动**：用基准测试数据说话

### 6.2 性能宣传原则

**禁止的表述**：
- ❌ "内存降低 10-40 倍"（未验证）
- ❌ "O(1) 预测查找已完成"（待验证）
- ❌ "性能提升 X 倍"（无数据支撑）

**推荐的表述**：
- ✅ "流式架构已实现"
- ✅ "预测查找架构已预留"
- ✅ "待基准测试验证实际性能"
