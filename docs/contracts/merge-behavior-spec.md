## 一、核心约束（已冻结）

### 1.1 插值方法

**当前版本支持两种插值方法**：

| 方法 | 说明 |
|------|------|
| `nearest` | 最近时间点插值 |
| `linear` | 线性插值 |

**接口预留**：系统已预留扩展接口，方便后续增加更多插值方法。

---

### 1.2 Nearest（最近点）语义

**定义**：找到时间轴上最接近目标时间的点，返回该点的值。

**行为规则**：

| 条件 | 行为 |
|------|------|
| 目标时间精确命中某点 | 返回该点值 |
| tolerance > 0，最近点在容差内 | 返回最近点值 |
| tolerance > 0，最近点超出容差 | 返回 NaN，记录警告 |
| tolerance = 0 | 必须精确匹配（距离为0），否则 NaN |
| tolerance 未设置（-1） | 无限制，返回最近点 |

**警告信息格式**：
```
Nearest point out of tolerance. Target: {时间}, Distance: {距离} us, Tolerance: {容差} us
```

---

### 1.3 Linear（线性插值）语义

**定义**：在目标时间前后两个点之间做线性插值。

**行为规则**：

| 条件 | 行为 |
|------|------|
| 目标时间在容差内精确命中某点 | 直接返回该点值 |
| 目标时间在容差内，没命中点 | 在前后两点间做线性插值 |
| 目标时间超出数据范围之前 | 返回最早点的值 |
| 目标时间超出数据范围之后 | 返回最晚点的值 |
| 目标时间超出容差 | 返回 NaN，记录警告 |

**Tolerance 约束**：前后两个边界点都必须在 tolerance 距离内。

**警告信息格式**：
```
Interpolation points out of tolerance. Before: {前点距离} us, After: {后点距离} us, Tolerance: {容差} us
```

---

### 1.4 Tolerance 优先级

```
tolerance = 0   → 必须精确匹配
tolerance > 0   → 在容差范围内查找/插值
tolerance 未设置（-1）→ 无限制
```

---

### 1.5 时间格式支持

**支持的格式**：

| 格式类型 | 输入示例 | 精度 |
|---------|---------|------|
| 点号毫秒 | `HH:MM:SS.fff` | 3位 |
| 点号微秒 | `HH:MM:SS.ffffff` | 6位 |
| 冒号毫秒 | `HH:MM:SS:fff` | 3位 |
| 冒号微秒 | `HH:MM:SS:ffffff` | 6位 |

**精度自动推断**：1-3位小数 → 毫秒，4-6位小数 → 微秒

## 二、引擎约束（已冻结）

### 2.1 文件处理约束

| 约束 | 说明 |
|------|------|
| 第一个文件 | 必须是基准时间轴，数据直接透传 |
| Fallback 文件 | 不能作为第一个文件，否则报错 |
| 输入文件数 | 至少 2 个文件 |

### 2.2 输出列名策略

| 场景 | 策略 |
|------|------|
| 所有列 | 直接使用原始列名，不添加文件名前缀 |
| 重复列名 | 直接保留，不做重命名 |

**示例**：
- 文件1：`Time, Temp, Pressure`
- 文件2：`Time, Sensor1, Sensor2`
- 输出：`Time, Temp, Pressure, Sensor1, Sensor2`

### 2.3 文件分类

| 类型 | 说明 |
|------|------|
| A 类（SequentialUniform） | 单增 + 均匀间隔 |
| B 类（SequentialCorrected） | 单增 + 不均匀间隔 |
| C 类（IndexedFallback） | 乱序，需要全加载后排序处理 |

## 三、CLI 约束（已冻结）

### 3.1 参数约束

| 参数 | 有效值 |
|------|-------|
| `-i, --interpolation` | `nearest`, `linear` |
| `-E, --engine` | `legacy`, `stream` |
| `-t, --tolerance` | 整数（微秒），0表示精确匹配 |
| 输入文件数 | 至少 2 个 |

### 3.2 帮助文案约束

- 输入文件：至少 2 个
- 不再支持 `none` 插值方法
