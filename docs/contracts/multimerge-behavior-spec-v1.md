## 一、输入文件数量约束

### 1.1 规则

| 约束 | 值 |
|------|-----|
| 最少输入文件数 | **2 个** |
| 最多输入文件数 | 无限制 |

### 1.2 理由

- 1 个文件无法合并
- 2 个及以上才需要对齐合并逻辑

### 1.3 错误信息

```
错误：至少需要两个输入文件
```

## 二、Tolerance 规则

### 2.1 有效值

| 值 | 含义 |
|-----|------|
| `-1`（未设置） | 无限制，返回最近点/在范围内插值 |
| `0` | 必须精确匹配，否则 NaN |
| `> 0` | 容差范围（微秒），超出返回 NaN |

### 2.2 tolerance 与插值方法的关系

**Nearest**：
- tolerance 检查最近点距离
- 最近点距离 ≤ tolerance → 返回该点值
- 最近点距离 > tolerance → 返回 NaN，记录警告

**Linear**：
- tolerance 检查前后两个边界点距离
- 两个边界点都在 tolerance 内 → 执行插值
- 任一边界点超出 tolerance → 返回 NaN，记录警告

### 2.3 警告信息格式

```
Nearest: "Nearest point out of tolerance. Target: {时间}, Distance: {距离} us, Tolerance: {容差} us"
Linear: "Interpolation points out of tolerance. Before: {前距离} us, After: {后距离} us, Tolerance: {容差} us"
```

## 三、Interpolation 插值规则

### 3.1 支持的方法

| 方法 | 说明 |
|------|------|
| `nearest` | 最近时间点插值 |
| `linear` | 线性插值 |

### 3.2 Nearest 语义

| 条件 | 行为 |
|------|------|
| 精确命中 | 返回该点值 |
| tolerance > 0，最近点在容差内 | 返回最近点值 |
| tolerance > 0，最近点超出容差 | 返回 NaN，记录警告 |
| tolerance = 0 | 必须精确匹配（距离为0），否则 NaN |
| tolerance = -1（未设置） | 无限制，返回最近点 |

### 3.3 Linear 语义

| 条件 | 行为 |
|------|------|
| 精确命中（某点） | 返回该点值 |
| 在容差内，未命中点 | 在前后两点间线性插值 |
| 超出数据范围之前 | 返回最早点的值 |
| 超出数据范围之后 | 返回最晚点的值 |
| 边界点超出 tolerance | 返回 NaN，记录警告 |

### 3.4 优先级

```
tolerance = 0   → 必须精确匹配
tolerance > 0  → 在容差范围内
tolerance = -1  → 无限制
```

## 四、输出表头规则

### 4.1 规则

直接使用原始列名，**不添加文件名前缀**。

### 4.2 示例

| 文件1表头 | 文件2表头 | 输出表头 |
|----------|----------|----------|
| Time | Time | Time |
| Temp | Sensor1 | Temp |
| Pressure | Sensor2 | Pressure |
| | Sensor3 | Sensor1 |

**输出**：`Time, Temp, Pressure, Sensor1`

### 4.3 重复列名

重复的列名直接保留，不做重命名。

## 五、错误/警告/取消语义

### 5.1 错误（Error）

**定义**：操作无法继续，需要终止

**触发条件**：
- 输入文件数 < 2
- 输入文件不存在
- 输入文件无法读取
- Fallback 文件作为第一个文件

**输出**：
```
错误：{具体原因}
合并失败
```

### 5.2 警告（Warning）

**定义**：操作可以继续，但结果可能不符合预期

**触发条件**：
- tolerance 超标，返回 NaN
- 数据文件与基准文件时间范围重叠
- 空值/NaN 数据

**输出**：
```
警告：{具体原因}
```

### 5.3 取消（Cancel）

**定义**：用户主动请求停止

**触发**：用户按 Ctrl+C 或调用取消接口

**输出**：
```
合并已取消
```

**特征**：
- 取消与失败严格分离
- 取消后可以重新开始
- 失败后需要排查原因
