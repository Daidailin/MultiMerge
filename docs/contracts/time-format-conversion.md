> **文档版本**: v1.0
> **创建时间**: 2026-03-21

## 一、支持的时间戳格式

### 1.1 格式列表

| 格式类型 | 输入示例 | 精度 | 输出格式 |
|---------|---------|------|---------|
| 冒号毫秒 | `12:30:45:123` | 毫秒（3位） | `12:30:45:123` |
| 冒号微秒 | `12:30:45:123456` | 微秒（6位） | `12:30:45:123456` |
| 点号毫秒 | `12:30:45.123` | 毫秒（3位） | `12:30:45:123` |
| 点号微秒 | `12:30:45.123456` | 微秒（6位） | `12:30:45:123456` |

### 1.2 分隔符规则

- **冒号格式**：`HH:MM:SS:ffffff`（秒与小数之间用冒号分隔）
- **点号格式**：`HH:MM:SS.ffffff`（秒与小数之间用点号分隔）

## 二、补零规则

### 2.1 时分秒部分（左补零）

| 输入 | 输出 |
|------|------|
| `1:2:3:5` | `01:02:03:005` |
| `12:30:45:123` | `12:30:45:123` |
| `0:0:0:000` | `00:00:00:000` |

### 2.2 小数部分（右补零至固定精度）

**毫秒精度（3位小数）**：
| 输入 | 输出 | 说明 |
|------|------|------|
| `12:30:45.1` | `12:30:45:100` | 1位→右补2零至3位 |
| `12:30:45.12` | `12:30:45:120` | 2位→右补1零至3位 |
| `12:30:45.123` | `12:30:45:123` | 已是3位，保持 |
| `12:30:45:4` | `12:30:45:004` | 冒号格式同样处理 |

**微秒精度（6位小数）**：
| 输入 | 输出 | 说明 |
|------|------|------|
| `12:30:45.1` | `12:30:45:100000` | 1位→右补5零至6位 |
| `12:30:45.1234` | `12:30:45:123400` | 4位→右补2零至6位 |
| `12:30:45.123456` | `12:30:45:123456` | 已是6位，保持 |

## 三、精度判断规则

### 3.1 自动精度推断

**规则**：
- 小数部分 **1-3 位** → 毫秒精度（`TimeResolution::Milliseconds`）
- 小数部分 **4-6 位** → 微秒精度（`TimeResolution::Microseconds`）
- 小数部分 **>6 位** → 截断至微秒精度（前6位）

### 3.2 内部存储单位

| 精度级别 | 内部存储 | 最大值 |
|---------|---------|-------|
| 毫秒 | `m_fraction` = 毫秒值 | 0-999 |
| 微秒 | `m_fraction` = 微秒值 | 0-999999 |

### 3.3 统一转换

所有输入格式在内部统一转换为 **纳秒** 进行计算：
```cpp
// 毫秒转纳秒
totalNs += m_fraction * 1000000LL;

// 微秒转纳秒
totalNs += m_fraction * 1000LL;
```

## 四、实现细节

### 4.1 统一真相来源

**修改前**：
- FileReader::parseLine 有独立的内联解析逻辑
- StreamMergeEngine 调用 TimeParser
- 两条解析路径不一致

**修改后**：
- TimeParser 同时支持冒号和点号格式
- FileReader::parseLine 改为调用 TimeParser::instance().parse()
- 所有时间解析统一入口

### 4.2 代码位置

| 文件 | 职责 |
|------|------|
| `TimeParser.cpp` | 注册4种格式解析器 |
| `TimeParser.cpp::parseDotFractionalTime` | 点号格式解析 |
| `TimeParser.cpp::parseColonFractionalTime` | 冒号格式解析 |
| `FileReader.cpp::parseLine` | 改为调用 TimeParser |

### 4.3 格式注册顺序

```cpp
registerBuiltInFormats() {
    // 1. 冒号毫秒
    m_formats.append("ColonMilliseconds", R"(^\d{1,2}:\d{1,2}:\d{1,2}:\d{1,3}$)", ...);
    // 2. 冒号微秒
    m_formats.append("ColonMicroseconds", R"(^\d{1,2}:\d{1,2}:\d{1,2}:\d{1,6}$)", ...);
    // 3. 点号毫秒
    m_formats.append("DotMilliseconds", R"(^\d{1,2}:\d{1,2}:\d{1,2}\.\d{1,3}$)", ...);
    // 4. 点号微秒
    m_formats.append("DotMicroseconds", R"(^\d{1,2}:\d{1,2}:\d{1,2}\.\d{1,6}$)", ...);
}
```

## 六、待收敛问题

### 6.1 TimePoint 输出格式

当前 TimePoint::toString() 默认输出冒号格式：
```cpp
// 默认格式：HH:MM:SS:SSS（毫秒）或 HH:MM:SS:SSSSSS（微秒）
result += ":" + QString::number(m_fraction).rightJustified(decimalPlaces, '0');
```

如需支持点号输出，需扩展 format 参数。

### 6.2 精度降级处理

当输入超过6位小数时，当前行为是截断而非四舍五入：
```cpp
// 超过6位，截断到6位
fraction = fraction / 1000LL;
```
