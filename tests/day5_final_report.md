## 2. tolerance 行为验证

### 2.1 测试设计

**输入**:
- tol_t1_base.txt: 基准 0.0s, 1.0s, 2.0s
- tol_t1_src.txt: 偏移 0.05s (50ms) → 0.05s, 1.05s, 2.05s

**测试**: t=0（精确匹配模式）下nearest行为

### 2.2 测试结果

| 引擎 | tolerance | 预期 | 实际 | 结果 |
|------|-----------|------|------|------|
| stream | -t 0 | NaN（50ms>0） | 5.0, 105.0, 205.0 | **FAIL** |
| legacy | -t 0 | NaN（50ms>0） | 5.0, 105.0, 205.0 | **FAIL** |

### 2.3 结论

**tolerance在两条引擎中都未真正生效**：
- CLI -t 0 被转换为 0μs（精确匹配）
- 但stream和legacy都返回了有效值
- 证实了代码审计的发现：stream完全未实现tolerance，legacy未传递tolerance参数
